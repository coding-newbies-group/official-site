3.变量​

  • 前置知识：在Python解释器中成功输入多种类型的值。
  • 学习目标：学会给变量赋值，使用变量。

3.1.变量​

上学的时候我们学过一些简单的方程式，比如：

a = 1

b = 2

我们将1赋值给a，2赋值给b;

x= a + b就得到了x = 3;

y = a * b就得到了y = 2。

很显然，我们只需要修改a的值，就可以得到新的 x 和 y 的值。

在编程中，也是一样的，我们可以修改 a 和 b 的值，代码中所有用到 a 和 b 值的地方就跟着变了。

我们称 a , b为变量，即用来装 值 的容器。

而这个等号=被称作 赋值操作符。

赋值，就是：变量在左边，值在右边，通过等号=连接。

在学习 “第一行代码“ 的时候，我们通过print("Hello World")将Hello World输出到了屏幕上。

根据上面的知识，我们就可以把Hello World赋值给一个变量，然后将这个变量所代表的值输出到屏幕上：

修改后的代码如下：

msg = "Hello World"
print(msg)

上面的代码，我们将Hello World赋值给了变量msg，然后通过print(msg)把变量msg所代表的Hello World输出到了屏幕上。

以后，当我们想修改输出到屏幕上的内容，只需要改变msg的值就可以了，即通过 再次赋值 的方式，让变量等于新的值。

比如：

msg = "Good Morning"
print(msg)

屏幕上就会输出Good Morning。

7.流程控制——判断结构 和 代码块​

  • 前置知识：算数操作符，布尔值。
  • 学习目标：学会使用if, elif, else，理解代码的层次结构，代码之间的从属关系。

现实中的很多流程，并不是像上一章那样，自上而下的顺序发展。

更多的流程是，根据 特定的 条件 进行 判断 ，再决定如何下一步如何进行的。比如：

在游乐园中，很多游玩项目会 根据年龄决定 你是否可以游玩，一些比较惊险刺激的项目，就不允许小朋友游玩。

下面，我们通过一下简单的数字之间的关系，来学习这节课的内容。

现在，我们想象一个场景，假如用户提供了一个数字，即让a等于一个数字，然后判断a与5的大小关系：

如果 a > 5，则屏幕上输出“a大于5”；

如果 a < 5，则屏幕上输出“a小于5”；

如果 a = 5，则屏幕上输出“a等于5”。

应该如何用代码实现呢？这时，我们就需要用使用判断结构，来实现：如果满足特定的条件，就执行特定的动作。

下面是实现上面功能的python代码：

a = 5
if a > 5:
    print("a大于5")
elif a < 5:
    print("a小于5")
else:
    print("a等于5")

下面我们来解释一下这个代码：

a = 5

我们先假设用户输入了数字5，然后我们把5赋值给了变量a；

7.1. if​

if a > 5:

if是如果的意思，是判断结构的开始，if后面是判定的条件：a > 5，然后以一个英文的冒号:结尾。

整行的意思就是，如果 a 大于 5，那么：

我们继续向下看代码：

if a > 5:
    print("a大于5")

print("a大于5")，即在屏幕上输出a大于5。

也就是，现在代码的意思是，如果 满足a > 5这个条件，就在屏幕上输出a大于5。

7.2.elif​

我们继续看接下来的代码：

elif a < 5:

elif是else if的意思，else表示其他，所以，else if也就是同一个事情的其他情况。

所以，上面的代码就成了，否则，如果 a < 5，那么：

也就是：继续对a的值进行比较

也就是：只有当 a 不满足 a > 5 的情况时，才会判断 a 是否小于5。

注意，这一行的结尾，还是一个英文冒号:

再接下来的代码：

elif a < 5:
    print("a小于5")

print("a小于5")，即在屏幕上输出a小于5。

所以，这一小段代码的意思就是：

否则，如果 a 小于 5，就在屏幕上输出 a小于5 。

7.3.else​

我们继续看最后一段代码：

else:
    print("a等于5")

前面理解了else是否则的意思，想必，这段的意思，大家就很容易“猜”到了。

它的意思是：

否则，在屏幕上输出 a等于5。

注意，这里不同于elif a < 5:，else的后面没有任何判断条件。

这就意味着，无论怎样，都会执行print("a等于5")这行代码。

到此，我们再来整体看一遍代码的意思：

a = 5
if a > 5:
    print("a大于5")
elif a < 5:
    print("a小于5")
else:
    print("a等于5")

将5 赋值给变量a

如果，a 大于 5，那么：

​ 在屏幕上输出 a大于5

否则，如果，a 小于 5，那么：

​ 在屏幕上输出 a小于5

否则：

​ 在屏幕上输出 a等于5

请原谅我在这个地方需要再啰嗦一下：

注意，请再细细理解一下else，else意味着，不满足前面的判断条件，才会执行后面的内容。

也就是，如果a > 5，打印 “a大于5”，假如不满足 a > 5，才会 接着 向下判断，如果满足a > 5，执行打印"a大于5"，整个过程就直接结束了。

而对于，最后的else，此时，也就不难理解为，因为else后面没有任何判断，意味着，当上面所有的条件都不满足，才会执行，也就是else后面，是一个“兜底“的动作。

为了验证我们是否理解上面的内容，猜猜下面代码运行的结果，会在屏幕上输出A, B, C, D哪些字母？请先自己分析。

a = 3
if a < 5:
    print("A")
elif a < 10:
    print("B")

if a < 5:
    print("D")
if a < 10:
    print("E")

答案是：

屏幕上会输出 A,D,E

下面是代码执行的效果：

细新的你一定已经发现，在VSCode页面，我们可以看到代码的左侧都会有行号，表示代码所在的行数，可以方便我们定位，交流具体的代码。

下面，我们一起分析一下，结果为什么是A,C,D：

首先， a = 3

我们来看2-5行的代码：

if a < 5:
    print("A")
elif a < 10:
    print("B")

因为a等于3，所以a小于5的条件是满足的，于是，代码print("A")执行了;

而，elif是否则，如果...

因此，即使a也满足后面的a小于10，

但是，并没有触发elif这个动作。

即，并没有轮到判断：如果 a 小于 10。

所以，print("B")更加不会执行。

所以，输出的结果只是A。

我们再来看7-10行的代码：

if a < 5:
    print("D")
if a < 10:
    print("E")

if a < 5: 和 if a < 10:是独立的两个if判断，两个判断条件并没有通过elif来连接。

稍微调整如下，在两段if结构中间加个空行就更容易理解了：

if a < 5:
    print("D")
    
if a < 10:
    print("E")

同样对于a = 3的情况，会分别判断a是否满足小于5，以及小于10的情况。

因此，a = 3, 就会既符合 a < 5,又符合 a < 10。

所以，屏幕上就依次输出了D和E。

所以，整段代码最终的输出结果会是A, D和E。

同时，细心的你还会发现，我们这段代码并没有用到else，也就是这个“兜底”不是必须的，我们根据条件做判断，做选择的时候，完全允许没有“兜底”的情况。

好，到此，if, elif, else我们就讲清楚了。

7.5.代码块​

通过对上面的图的观察，我们发现一个规律：

  1. print(xxx)的前一行的代码会以英文的冒号:结尾；
  2. 每个print(XXX)与前一行代码是有缩进关系的，即print(xxx)的左侧与if，elif的左侧相比，向右缩进了4个空格的位置。

这表明了他们的从属关系，他们组成了一个代码块。

当然，冒号后面保持相同缩进的代码，都属于同一个代码块。

比如：

print("A"), print("B"), print("C") 都是 一家人 ，当 a < 5 条件成立时，会依次执行。

而print("D")从属于 a > 5 ，所以，条件不成立，不会执行。

而且，冒号后面，还必须有从属，否则会报错：

把print("D")删除后，运行代码会报错，并提示：

IndentationError: expected an indented block after 'if' statement on line 8

最后，提示两点：

  1. 第9行因为在冒号后面没有代码，VSCode会有一个红色的波浪符的提示，VSCode会对可能的代码错误，给与我们提示。
  2. 实际上，我们在VSCode中代码缩进时，通常会使用Tab键来完成这个缩进，VSCode会自动帮我们把Tab转换为输入4个空格。

7.7.总结​

这节课，我们：

  • 学习了if，根据条件做判断；
  • 学习了elif，是else + if的意思，else表示否则，即不满足前面的条件时才会轮到else后面的if；
  • 学习了else，是一个没有后续if的else，用来“兜底“；
  • 将顺序结构，判断结构放在一起，观察了流程控制；
  • 学习了代码块的知识：通过4个空格的向右缩进表示代码的从属关系；
  • 敢于面对代码运行出现错误，将问题解决掉就好，没什么大不了。
